TRAINING THE MODEL

------------------------------------------------------------

Run the following command from the project root directory:

python train\_segmentation.py



During training:

\- Model checkpoints are saved automatically.

\- Training \& validation metrics (Loss, IoU, Dice, Accuracy) are logged.



TESTING \& INFERENCE

------------------------------------------------------------

To evaluate the trained model and generate predictions:

python test\_segmentation.py



This will:

\- Run inference on test images.

\- Save predicted segmentation masks in the outputs/ folder.

\- Compute IoU and other evaluation metrics.

PERFORMANCE SUMMARY

------------------------------------------------------------

\- Validation IoU: ~0.31

\- Validation Dice: ~0.44

\- Validation Accuracy: ~71%



CHALLENGES \& SOLUTIONS

------------------------------------------------------------

\- Challenge: Missing segmentation masks for some test images caused evaluation errors.

\- Solution: Dataset was manually verified and corrected to ensure consistency. Evaluation was resumed successfully after fixing paths.



OBSERVATIONS \& FAILURE CASES

------------------------------------------------------------

\- Model performs well on clear road boundaries.

\- Struggles in: low-light conditions, highly cluttered scenes, and ambiguous terrain textures.



NOTES FOR EVALUATORS

------------------------------------------------------------

\- Model trained without GPU.

\- DINOv2 backbone loaded via torch hub.

\- All paths are configurable.
